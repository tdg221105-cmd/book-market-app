# Android Book Trading Application

## 1. Project Overview
This project is an Android mobile application designed for buying and selling books through a multi-role system. The application supports three different user roles: **Buyer**, **Seller**, and **Shipper**, each with its own interface and functionalities. The system aims to provide a convenient and efficient platform for book trading, order management, and delivery coordination.

The application is developed using modern Android technologies and follows a client–server architecture to ensure scalability and real-time data synchronization.

## 2. User Roles and Functionalities

### 2.1 Buyer (Customer)
Buyers can use the application to:
- Browse available books by category or seller  
- View detailed information about books (price, description, availability)  
- Place orders and manage purchases  
- Track order status and delivery progress  
- View order history  

---

### 2.2 Seller (Book Owner)
Sellers can use the application to:
- Add new books for sale  
- Update book information such as price, quantity, and description  
- Manage book inventory  
- View and process customer orders  
- Confirm or cancel orders  

---

### 2.3 Shipper (Delivery Staff)
Shippers can use the application to:
- View assigned delivery orders  
- Update delivery status (picked up, in transit, delivered)  
- Manage delivery tasks efficiently  
- Confirm successful deliveries  

---

## 3. Main Features
- Multi-role user system with separate interfaces  
- Book listing and management  
- Order placement and tracking

- Real-time data updates between users

## 4. Technologies Used
- **Android Studio** – Integrated Development Environment (IDE)

- **Firebase** – Authentication, real-time database, and cloud services  
- **Ktor** – Backend framework for handling APIs and server-side logic
